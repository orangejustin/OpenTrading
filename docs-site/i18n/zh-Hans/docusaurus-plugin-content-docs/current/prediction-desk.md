# 预测台

每个个股页都运行同一条流水线。本页解释每个阶段以及其背后的命令。

## 1 · 预测者

相互独立的观点，每个都来自*不同的*信息：

| 命令 | 观点 | 方法 |
|---|---|---|
| `ot decide` | 规则引擎 | CALL/PUT/NO-ACTION + 一个区间计划 |
| `ot quant` | 逻辑回归 P(up) + 区间锥 | 免密钥逻辑回归，经样本外（OOS）把关 |
| `ot forecast` | TimesFM 2.5 分位数锥 | 基础模型（需主动选择安装） |
| `ot options` | 做市商 Gamma（GEX） | 每个行权价的带符号 gamma × OI，来自 CBOE 期权链 |
| `ot poly` | 群体赔率 | 用于宏观事件门控的 Polymarket 价格 |
| `ot hl` | 永续资金费率与 OI | Hyperliquid —— 加密杠杆的账单 |
| `ot whales` | 链上流向 | 已打标签钱包的 ETH 余额 + 增量 |

关键在于*独立性*：不同的数学方法，不同的数据。一个来自统计的锥和一个来自做市商对冲的墙如果落在同一个价格上，那就是一种可交易的结构，而这是任何一方单独都无法呈现的。

## 2 · 融合

- **共识条** —— 每个分析师对应一个色块；一旦有两个产生分歧，判决就翻转为 **STAND ASIDE（观望）**。分歧本身就是信息。
- **共振价位梯** —— 交易台给出的所有价位，汇集在同一条价格轴上。由 **2 个以上方法**共同点名的行（`×2`）才是唯一重要的。

## 3 · 辩论

```bash
ot debate NVDA           # 3 LLM calls on a frozen evidence pack
```

三次调用运行在一个确定性的证据包之上：

1. **多头** —— 最强而诚实的做多理由。
2. **空头** —— 最强的做空/回避理由，*并且*它必须直接攻击多头最有力的论点。
3. **裁判** —— 对照证据权衡双方，并**做出明确判决**：一个 5 档判决，包含置信度、入场、一个强制性的失效价，以及一个时间止损。

默认情况下，多头和空头运行在*不同的*引擎上（视角多样性，Condorcet 式）；你也可以让三个角色都运行在同一个引擎上。每一个判决都会自动记入日志以便打分。

:::info 确定性的 SOP
证据包在任何模型运行*之前*就由脚本构建并冻结。模型只负责读取；它们从不抓取数据或采取行动。相同的输入 → 相同的证据包 → 可复现。
:::

## 4 · 学习环

```bash
ot reflect grade         # grade calls that have ripened
ot reflect stats         # hit-rate by action / grade / market
ot reflect lessons       # the lessons injected into the next judge
```

每一个已给出的判断稍后都会对照实际发生的情况来打分 —— 对还是错、相对 SPY 的收益（alpha）、最大不利偏移、失效价是否被突破 —— 而这些经验教训会反馈到未来的裁判提示词中。关于一个判断究竟在何时、以何种方式被评分，参见
**[方法论](./methodology.md)**。
