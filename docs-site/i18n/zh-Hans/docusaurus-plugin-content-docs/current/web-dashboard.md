# Web 仪表盘

```bash
ot web                   # → http://127.0.0.1:8787
```

仪表盘是交易台的前端。它在本地运行，读取你的持仓清单，并在未配置 LLM 引擎时优雅地降级为免密钥的数据面板。

![OpenTrading Web 仪表盘](/img/web-dashboard.png)

## 读懂一个个股页（从上到下）

1. **共识条** —— 每个分析师（engine、quant、TimesFM、AI、debate）对应一个色块，颜色反映其倾向。判决由这些投票计算得出：当有
   2 个以上一致且无人反对时为 **CONSENSUS LONG/SHORT**，没有多数时为 **NEUTRAL**，报告的分析师太少时为
   **THIN**，而一旦任意两个分析师指向相反方向，就立刻变为 **⚠ STAND ASIDE（观望）**。
2. **图表 + 墙 + 锥** —— 价格图，叠加了做市商的 **Call/Put 墙**和预测锥。墙的 **DTE 窗口可调**，可从共振价位梯的表头（`0DTE / 5 / 7 / 14 / 30 / 45 / 60 / 90`）选择 ——
   盯日内的钉住风险（pin risk）就选 0DTE，看波段结构就选更长的窗口。价位梯会显示该窗口下的净 GEX，随着你把窗口放宽，它也会随之变化。
3. **多空对决** —— 已给出的判断：一个带有入场、失效价和时间止损的 5 档判决。参见 [辩论](./prediction-desk.md)。
4. **共振价位梯** —— 交易台给出的每一个价格位都汇集在同一条轴上；由 **2 个以上相互独立的方法**共同点名的行（标记为
   `×2`）才是真正重要的。
5. **AI 分析** —— 一个有依据的第二意见：摘要、操作建议、情绪指标、狙击价位和风险。

## 切换引擎

顶部有一个**模型下拉框**。它控制 AI 分析和新闻分析面板；结果会按 (ticker, engine, model) 分别缓存。

**辩论**有它自己的 `engines` 芯片 —— 点击在三种模式间循环：

- **多引擎**（默认） —— 多头和空头运行在*不同的*引擎上以获得视角多样性；裁判则用 Claude。
- **单引擎** —— 三个角色全部运行在你在顶部选的那个引擎·模型上。
- **自选角色** —— 出现三个小下拉框：为多方、空方、裁判分别选择引擎·模型（留在 *auto* 则用台面默认）。适合 A/B 测试裁判、或让两个模型互相对垒。

切换不消耗 token；新的分配在下一次运行时生效。注意 `Claude Code · default`（以及 Codex 的 `default`）指的是*你 CLI 自己的默认模型* —— 可在 CLI 里用 `/model` 更改，或在这里直接固定一个。

## English / 中文

顶部的一个 **中文 / EN** 开关（或 `?lang=zh`）会把整个 UI *以及*模型的输出在英文和简体中文之间切换 —— 同一个交易台既可用于美股账本，也可用于 A 股 / 港股账本。

## Learn 标签页

`#/learn` 是内置的教科书。个股页上的每个模块都有一个 **?** 帮助标签，点击后会跳转到对应的讲解 —— 共振价位梯、预测锥、做市商
Gamma、聪明钱、校准 —— 每一个都配有一个真实案例的完整演算。当某个数字含义不明显时，就从那里开始看。

永续合约、**链上巨鲸流向**、引擎诊断，以及 Polymarket 群体赔率。
