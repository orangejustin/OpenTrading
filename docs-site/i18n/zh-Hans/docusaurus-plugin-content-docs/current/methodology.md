# 方法论

交易台如何给自己打分，以及有几个让人意外的数字究竟从何而来。完整的参考文档在仓库里：[**docs/METHODOLOGY.md**](https://github.com/orangejustin/OpenTrading/blob/main/docs/METHODOLOGY.md)。

## 校准 —— 一个判断在什么时候被评分？

```bash
ot reflect grade --days 5
```

一个判断一旦**存在 ≥ N 个交易日**（默认 **N = 5**，与辩论的 5 日波段视野相匹配）就会被评分。周一做出的判决，最早要到下一个周一才可评分。评分是**幂等的** ——
一个判断只评分一次，然后永远带着它的结果。

## CALL/PUT 是针对第 0 天还是第 +1 天？

这取决于该判断*做出*时所用的视野，它以 `time_stop_days` 的形式记录在每一行上：

- **0DTE**（`ot decide --dte 0`） —— 一个*当日*判断，按当天的走势评分。
- **波段**（`--dte 5`，辩论的默认值） —— 从*下一个*交易日入场，并在满 5 个交易日时评分。

交易台从不把两者混在一起：每一行都按它自己的视野来评分。"时间止损"就是"方向看对了、但价格迟迟没动"开始被判为错误的那个临界点。

## 两条诚实规则

1. **种子行被排除在外。**日志中自带了几个引导用的示例（`source: "seed"`），这样交易台在第一天就不是空的 ——
   但它们从不计入战绩或经验教训。
2. **它评的是交易台，而不是你的盈亏（P&L）。**这张表是交易台*自己*的判断历史（辩论 + 引擎解读），目前是**全局的 ——
   不按账本或账户拆分**。一个"A 股"行是交易台读过的某个中国标的，而不是任何人已实现的真实交易。

校准加权的融合 —— 即裁判会更信任一个被验证过的分析师 —— 要等到**约 30 个真实的已评分判断**之后才会启动。在那之前，个位数的单元格只是轶事，而非统计。

## 做市商 Gamma —— 为什么会有两个"Call 墙"？

同一个标的可能显示两个 Call 墙，因为它们是在**不同的 DTE 窗口**上计算的：

| 位置 | 窗口 | 它回答的问题 |
|---|---|---|
| 宏观与资金流卡片 | ≤ 7 DTE | 本周的钉住风险 |
| 个股价位梯 / 图表 | ≤ 30 DTE | 波段视野的墙 |

两者都是正确的 —— 它们回答的是不同的问题。图表会把它的墙标注为 `≤30D`，这样就不会有歧义。

## 保留策略

决策日志是**仅追加（append-only）**的，会被保留下来（会变的只是结果）。会随时间被清除的是 `data/news-log/` 下的新闻归档 ——
而不是决策日志。
